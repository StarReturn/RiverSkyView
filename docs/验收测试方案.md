# 本地项目桌面管理系统验收测试方案

> 适用技术基线：[`技术方案.md`](./技术方案.md)  
> 实施依据：[`CLAUDE实施方案.md`](./CLAUDE实施方案.md)  
> 文档版本：1.0  
> 更新日期：2026-07-15

## 1. 验收目标

验证本地项目桌面管理系统在 Windows 10/11 x64 环境中满足功能、安全、数据一致性、性能和安装交付要求，并确认以下核心承诺成立：

- 不移动或删除用户项目文件。
- 不实时监听或递归扫描整个项目目录。
- 不覆盖已有 `AGENTS.md` 和 `CLAUDE.md` 内容。
- 项目日志可被 Codex/Claude Code按协议创建并由应用正确同步。
- 服务器资料正文不会以明文落入 SQLite、日志或普通应用文件。
- 中文、空格和常见特殊字符路径可以正常使用。
- 安装包可以在干净 Windows 环境安装和运行。

## 2. 验收准入条件

开始正式验收前必须满足：

- 已生成候选 NSIS 安装包。
- `pnpm-lock.yaml`、`Cargo.lock` 和数据库 migrations 已纳入版本控制。
- 以下命令全部成功：

```powershell
pnpm install --frozen-lockfile
pnpm typecheck
pnpm lint
pnpm test
cargo fmt --manifest-path src-tauri/Cargo.toml --check
cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets -- -D warnings
cargo test --manifest-path src-tauri/Cargo.toml
pnpm tauri build
```

- 不存在已知的数据损坏或敏感信息泄漏问题。
- 测试使用的密码、令牌和私钥均为虚构测试数据。

## 4. 测试数据准备

### 4.1 项目目录

准备以下目录：

```text
D:\PM-Acceptance\普通项目
D:\PM-Acceptance\中文 项目 (A&B)#1
D:\PM-Acceptance\已有指令文件
D:\PM-Acceptance\损坏管理块
D:\PM-Acceptance\大目录
D:\PM-Acceptance\只读项目
D:\PM-Acceptance\项目外部
```

已有指令文件内容：

`AGENTS.md`：

```markdown
# Existing Codex Rules

- Keep this exact line.
```

`CLAUDE.md`：

```markdown
# Existing Claude Rules

- Keep this exact line.
```

大目录至少包含：

- 一个拥有 5,000 个直接子项的目录。
- 一个多层嵌套目录。
- 中文、空格、`&`、括号和 `#` 文件名。
- 指向项目外部的目录联接或符号链接。

### 4.2 预览文件

- 正常 UTF-8 Markdown。
- 包含本地相对图片的 Markdown。
- 超过默认 2 MiB 的 Markdown。
- 无效 UTF-8 文本。
- 包含脚本、事件处理器和 `javascript:` 链接的 Markdown。
- PNG、JPEG、GIF、WebP、BMP、SVG。
- 带脚本的恶意 SVG。
- 损坏图片和超大图片。
- 普通二进制文件。

### 4.3 项目日志

准备：

- 同一天的 Codex 与 Claude 合法日志各两份。
- `completed`、`failed`、`blocked` 各一份。
- 缺少 Front Matter 的日志。
- 无效日期日志。
- 未知 agent/status 日志。
- Front Matter 正常但正文含恶意 HTML 的日志。
- 共 10,000 个小型合法日志的性能数据集。

### 4.4 服务器资料

测试明文统一使用易搜索标记：

```text
PM_TEST_SECRET_7f31a9_DO_NOT_USE
```

准备：

- 小型 UTF-8 TXT。
- 中文 TXT。
- 多行服务器连接资料。
- 空文件。
- 较大但低于产品限制的 TXT。

验收完成后清理这些虚构测试资料。

## 5. 优先级与通过标准

| 优先级 | 含义 | 通过要求 |
|---|---|---|
| P0 | 数据安全、敏感信息、应用核心不可用 | 必须全部通过 |
| P1 | 核心需求和主要异常路径 | 必须全部通过 |
| P2 | 次要体验、兼容性和提示问题 | 允许有经确认的已知限制 |

最终通过条件：

- P0 通过率 100%。
- P1 通过率 100%。
- 自动化测试全部通过。
- P2 未通过项有明确影响、规避方式和修复计划。
- 不存在应用导致的项目文件丢失、指令文件截断或资料明文泄漏。
