> 文档版本：1.0  
> 更新日期：2026-07-15  
> 技术基线：[`技术方案.md`](./技术方案.md)  
> 实施基线：[`CLAUDE实施方案.md`](./CLAUDE实施方案.md)  
> 验收基线：[`验收测试方案.md`](./验收测试方案.md)

## 1. 设计定位

界面定位为“高效率 Windows 本地项目工作台”，核心不是展示大量统计数据，而是让用户在最少操作内完成：

1. 找到项目。
2. 打开项目目录、终端或 CLI 智能体。
3. 查看项目文件和工作日志。
4. 安全访问服务器资料。

整体体验应接近成熟的桌面开发工具：信息密度适中、操作入口稳定、键盘友好、错误可恢复。避免将首页设计成营销式大卡片仪表盘，也避免为了视觉效果牺牲路径、时间和操作按钮的可读性。

## 2. 设计原则

### 2.1 快速到达

- 常用项目在启动后立即可见。
- 搜索框获得自然焦点，输入即可过滤。
- 打开 CMD、Codex、Claude Code 和资源管理器不超过两次点击。
- 常用动作同时提供右键菜单和键盘快捷键。

### 2.2 安全动作可辨识

- 项目文件区域不出现移动和删除入口。
- “从系统移除”明确说明不会删除磁盘目录。
- 资料库正文默认遮罩，不因选中列表项自动暴露。
- 永久清除、批量移除等动作使用危险色和二次确认。

### 2.3 路径优先

项目路径是判断项目身份的重要信息。列表和详情页均应展示完整路径或可查看完整路径的省略文本，不以模糊图标代替路径。

### 2.4 渐进呈现

- 首页只呈现项目选择和启动所需内容。
- 文件操作在文件树上下文中出现。
- 活动热力图和日志位于项目活动工作区。
- 技术错误默认转换为可理解中文，详细信息按需展开。

### 2.5 桌面优先

- 设计基准窗口为 1440 × 900。
- 最小窗口建议为 1024 × 680。
- 不为手机布局设计，但应支持窄窗口下导航折叠和面板收起。

## 4. 全局应用框架

### 4.1 主布局

采用“左侧固定导航＋右侧主内容”的桌面布局。

```text
┌────────────────────────────────────────────────────────────────────┐
│ Windows 原生标题栏                                                 │
├──────────────┬─────────────────────────────────────────────────────┤
│              │ 页面标题 / 面包屑                     页面主操作    │
│  项目        ├─────────────────────────────────────────────────────┤
│  收藏        │                                                     │
│  最近使用    │                                                     │
│              │                  页面内容                           │
│  服务器资料  │                                                     │
│              │                                                     │
│  回收站      │                                                     │
│              │                                                     │
│  设置        │                                                     │
├──────────────┴─────────────────────────────────────────────────────┤
│ 状态提示：数据库正常 · 3 个项目 · 最近同步 14:32                  │
└────────────────────────────────────────────────────────────────────┘
```

第一版使用 Windows 原生标题栏，不自绘窗口关闭、最大化和最小化按钮，以降低拖动、缩放、多显示器和辅助功能问题。

### 4.2 左侧导航

- 展开宽度：216 px。
- 折叠宽度：64 px。
- 图标大小：18 px。
- 导航项高度：40 px。
- 当前项使用浅主色背景、主色图标和较高字重。
- 回收站不使用危险色常驻强调，只有存在已移除内容时显示数量徽标。
- 窗口宽度低于 1180 px 时允许折叠；用户选择需要记忆。

### 4.3 页面标题栏

内容区域顶部设置 64 px 页面标题栏：

- 左侧：标题、面包屑、可选状态。
- 右侧：页面级主要操作和更多菜单。
- 每页最多一个实心主按钮，其余使用普通或文本按钮。

### 4.4 底部状态栏

高度 28 px，显示低干扰状态：

- 数据库状态。
- 当前项目目录可用状态。
- 当前日志同步状态。
- 后台操作进度。
- 应用版本入口。

普通成功不弹全局通知；长任务完成、错误或需要用户处理时才使用通知。

## 5. 视觉设计系统

### 5.1 视觉风格

采用冷静、克制的开发工具风格：

- 中性灰作为大面积背景。
- 蓝色作为主要操作色。
- 绿色只用于成功、可用状态和活动热力图。
- 红色只用于危险操作和严重错误。
- 卡片边界依靠细描边和轻微背景层级，不使用浓重阴影。

### 5.2 字体

```css
font-family:
  "Segoe UI Variable",
  "Microsoft YaHei UI",
  "Segoe UI",
  sans-serif;
```

代码、路径和日志时间使用：

```css
font-family: "Cascadia Mono", "Consolas", monospace;
```

字号建议：

| 场景 | 字号 | 字重 |
|---|---:|---:|
| 页面标题 | 20 px | 600 |
| 区块标题 | 16 px | 600 |
| 正文/表格 | 14 px | 400 |
| 辅助说明 | 12 px | 400 |
| 路径/状态栏 | 12–13 px | 400 |

### 5.3 颜色令牌

亮色主题：

| 令牌 | 色值 | 用途 |
|---|---|---|
| `--bg-app` | `#F5F7FA` | 应用背景 |
| `--bg-surface` | `#FFFFFF` | 面板和表格背景 |
| `--bg-subtle` | `#F8FAFC` | 次级区域 |
| `--border` | `#E2E8F0` | 常规边框 |
| `--text-primary` | `#172033` | 主要文字 |
| `--text-secondary` | `#64748B` | 辅助文字 |
| `--primary` | `#2563EB` | 主要操作 |
| `--primary-hover` | `#1D4ED8` | 主色悬停 |
| `--success` | `#168A50` | 成功/可用 |
| `--warning` | `#B76E00` | 警告 |
| `--danger` | `#DC2626` | 危险/失败 |

暗色主题作为首个正式版本范围，跟随系统并允许手动选择。暗色背景不得使用纯黑，建议从 `#111827`、`#182033`、`#243047` 建立层级。

### 5.4 间距与圆角

- 基础间距单位：4 px。
- 常用间距：8、12、16、24、32 px。
- 输入框和按钮圆角：6 px。
- 面板圆角：8 px。
- 弹窗圆角：10 px。
- 表格行高：48 px；紧凑模式可降至 40 px。
- 普通按钮高度：32 px；主要表单按钮高度：36 px。

### 5.5 图标

使用统一的线性图标体系。图标只作辅助，项目移除、永久清除、复制密码等关键动作必须保留文字或 Tooltip，不能仅靠图标表达。

## 6. 项目列表页

### 6.1 页面目标

让用户快速搜索项目，并直接执行打开目录、终端或智能体操作。

### 6.2 页面布局

```text
┌────────────────────────────────────────────────────────────────────┐
│ 全部项目                                      ＋ 添加项目           │
├────────────────────────────────────────────────────────────────────┤
│ 🔍 搜索项目名称或路径...   [全部] [收藏] [最近]     排序：最近活动  │
├────────────────────────────────────────────────────────────────────┤
│ □  ★  项目名称        项目路径              最近活动      操作     │
│ □  ☆  SiamFeast       D:\Project...         今天 14:32    [CMD] ⋯  │
│ □  ★  户型生成器      D:\EarnMoney...       昨天 21:10    [CMD] ⋯  │
│ □  ☆  文档工具        D:\Tools...           暂无活动      [CMD] ⋯  │
├────────────────────────────────────────────────────────────────────┤
│ 已选择 2 项      [取消选择]                    [从系统移除]          │
└────────────────────────────────────────────────────────────────────┘
```

### 6.3 表格列

| 列 | 规则 |
|---|---|
| 选择 | 仅批量模式使用，表头支持全选当前结果 |
| 收藏 | 星标按钮，点击不进入详情 |
| 项目名称 | 主文字；点击进入项目详情 |
| 项目路径 | 等宽字体，单行省略，悬停显示完整路径 |
| 添加时间 | 默认可隐藏，在列设置中开启 |
| 最近活动 | 相对时间＋完整时间 Tooltip |
| 状态 | 仅目录不可用或规则损坏时显示明显状态 |
| 操作 | 固定 CMD 快捷按钮＋更多菜单 |

更多菜单内容：

- 打开项目详情。
- 打开资源管理器。
- 打开 CMD。
- 启动 Codex。
- 启动 Claude Code。
- 修改名称。
- 检查日志规则。
- 从系统移除。

“从系统移除”文案不得简写成“删除项目”。确认框明确写明：不会删除磁盘中的项目目录和文件。

### 6.4 列表状态

空状态：

```text
尚未添加项目
选择一个本地项目目录，之后可以从这里快速打开终端和智能体。
[添加第一个项目]
```

搜索无结果：保留搜索框和筛选条件，提供“清除筛选”，不显示添加项目引导。

目录不可用：行保持可见，路径旁显示警告图标；打开操作禁用或先提示重新定位，软移除仍可使用。

## 7. 添加项目流程

采用三步模态流程，宽度约 640 px。

### 第一步：选择目录

- 目录输入框只读。
- “选择目录”调用系统目录选择器。
- 显示规范化后的完整路径。
- 立即检查是否重复、是否存在、是否可写。

### 第二步：项目信息

- 项目名称默认使用目录名，可修改。
- 可选收藏。
- 显示默认启动配置摘要。

### 第三步：变更确认

```text
将向项目目录写入以下管理文件：

✓ 创建或更新 AGENTS.md
✓ 创建或更新 CLAUDE.md
✓ 创建 pm_log 目录
✓ 创建 .project-manager.json

已有文件不会被覆盖，只更新带 project-manager 标记的管理块。

[上一步]                                    [确认添加]
```

如果检测到损坏管理块，禁止确认，并提供“在资源管理器中打开”和“复制错误详情”。

添加过程显示逐项状态；任一步失败时展示已回滚内容，不以无上下文的“操作失败”结束。

## 8. 项目详情页

### 8.1 项目头部

```text
← 全部项目 / SiamFeast
D:\Project Documentation\siamfeast

[资源管理器] [CMD] [Codex] [Claude]                         [更多]
```

- 项目名称可通过更多菜单修改。
- 路径使用等宽字体，可复制。
- 目录不可用时头部显示警告条，并禁用依赖目录的操作。

### 8.2 工作区切换

项目详情使用两个一级页签：

- 文件。
- 活动。

把热力图和完整时间线放入“活动”页签，避免文件预览时出现拥挤三栏界面。

### 8.3 文件工作区

```text
┌──────────────────────┬─────────────────────────────────────────────┐
│ 文件                  │ src/views/ProjectList.vue        [复制路径] │
│ 🔍 筛选当前目录       ├─────────────────────────────────────────────┤
│ ▾ src                │                                             │
│   ▾ views            │              只读文件预览                   │
│     ProjectList.vue  │                                             │
│   ▸ components       │                                             │
│ ▸ src-tauri          │                                             │
│   README.md          │                                             │
└──────────────────────┴─────────────────────────────────────────────┘
```

- 左侧文件树默认宽度 300 px，可在 240–480 px 间拖动。
- 拖动宽度需要记忆。
- 文件树顶部提供当前目录筛选、刷新和显示隐藏文件菜单。
- 右侧没有选择文件时显示项目概览：路径、最近日志、快捷启动入口。
- 预览顶部显示文件名和相对路径，并明确标注“只读”。
- Markdown 预览提供“渲染/源码”切换，但源码模式同样只读。
- 图片工具栏放在预览上方居中，显示当前缩放比例。
- 二进制和超限文件显示文件信息及“在系统中打开/定位”，不显示空白区域。

文件树右键菜单：

目录：

- 展开/收起。
- 在资源管理器中打开。
- 复制绝对路径。

文件：

- 预览。
- 复制文件。
- 复制绝对路径。
- 在资源管理器中定位。

不提供移动、重命名和删除。

### 8.4 活动工作区

```text
┌────────────────────────────────────────────────────────────────────┐
│ 项目活动             [全部智能体⌄] [全部状态⌄] [刷新日志]          │
├────────────────────────────────────────────────────────────────────┤
│ 过去一年   12 个任务 · 9 完成 · 1 失败 · 2 阻塞                   │
│                                                                    │
│      GitHub 风格 52 周活动热力图                                   │
│                                                                    │
├──────────────────────────────────┬─────────────────────────────────┤
│ 时间线                           │ 日志详情                        │
│ ● 14:32 Codex 完成项目搜索       │ # 完成项目搜索                  │
│ ● 11:05 Claude 修复预览安全      │ 任务目标 / 完成内容 / 验证结果 │
│ ○ 昨天 Codex 阻塞：缺少依赖      │                                 │
└──────────────────────────────────┴─────────────────────────────────┘
```

- 热力图横向占满页面，默认最近 52 周。
- 色阶使用五级绿色，中性空白表示无活动，不把失败映射为红色热力格。
- 点击日期将时间线过滤到当天；再次点击或清除标签恢复全部。
- 时间线左侧状态图标区分完成、失败、阻塞。
- 时间线和详情默认 40/60 分栏，可拖动。
- 详情 Markdown 必须经过与文件预览相同的净化流程。
- 格式异常日志显示黄色状态和解析错误摘要，仍允许查看原始只读文本。
- 顶部显示“上次同步时间”，明确日志不是实时监听结果。

## 9. 项目回收站

使用简化表格展示名称、原路径、移除时间和操作。

操作：

- 恢复。
- 在资源管理器中打开原目录（目录存在时）。

项目回收站不提供“永久删除项目目录”。如果未来需要清理数据库历史记录，文案必须是“清除管理记录”，并继续明确不触碰项目目录。

## 10. 服务器资料库

### 10.1 页面布局

采用左列表、右详情编辑的双栏结构。

```text
┌──────────────────────────┬─────────────────────────────────────────┐
│ 服务器资料    [+ 新建]   │ 生产服务器 A                [编辑] [⋯] │
│ 🔍 搜索名称或标签        │ 最后更新：2026-07-15 14:32              │
│                          ├─────────────────────────────────────────┤
│ ● 生产服务器 A           │                                         │
│   production · ssh       │              内容已遮罩                 │
│                          │                                         │
│ ● 测试服务器             │          [按住显示]  [复制内容]         │
│   testing                │                                         │
│                          │ 此内容使用 Windows 当前用户加密保护。    │
└──────────────────────────┴─────────────────────────────────────────┘
```

- 左栏默认 320 px。
- 列表只显示名称、标签和更新时间，不显示正文摘要。
- 新建按钮为本页唯一实心主按钮。
- 支持“新建资料”和“导入 TXT”两个入口。
- 列表选中只加载元数据，不能自动解密和显示正文。

### 10.2 查看模式

- 默认显示遮罩占位，不用星号模拟真实长度。
- 采用“按住显示”作为快速查看，松开立即遮罩。
- 同时提供显式“显示内容”开关，开启后显示倒计时并在超时、切换条目、最小化或离开页面时重新遮罩。
- 复制内容时显示“已复制，将在 30 秒后清理剪贴板”。
- 如果用户在此期间复制其他内容，状态更新为“剪贴板已改变，不再清理”。

### 10.3 编辑模式

- 名称和标签位于正文上方。
- 正文使用等宽字体多行编辑器。
- 保存和取消固定在底部操作栏。
- 离开未保存编辑时必须确认。
- 保存过程中禁用重复提交并显示加密写入进度。
- 保存失败时保留当前编辑内容供用户重试，同时明确旧密文未被覆盖。

### 10.4 删除与恢复

- “移至资料回收站”使用普通确认，不在主界面显示永久清除。
- 资料回收站提供恢复和永久清除。
- 永久清除使用危险对话框，要求再次确认资料名称。
- 界面说明“永久清除应用保存的密文”，不宣称完成物理不可恢复擦除。

### 10.5 DPAPI 限制提示

资料库首次使用时展示一次可关闭说明：

```text
服务器资料使用当前 Windows 用户加密保护。
将应用数据文件复制到其他电脑或其他 Windows 用户下，通常无法直接解密。
请不要把本功能视为跨设备密码备份工具。
```

设置页和资料库帮助入口应能再次查看。

## 11. 设置页

设置页使用左侧分类、右侧表单，避免一个超长页面。

### 常规

- 主题：跟随系统/亮色/暗色。
- 开机启动。
- 最小化到托盘。
- 全局唤起快捷键。

### 启动与终端

- 默认终端配置。
- CMD、PowerShell、Codex、Claude Code、VS Code、Cursor 可执行文件检测。
- 自定义启动配置管理。
- 参数模板预览，不提供直接 Shell 字符串输入框。

### 文件与预览

- 是否显示隐藏文件。
- Markdown 最大预览大小。
- 图片最大预览大小。
- 是否允许经过确认后加载远程资源，默认关闭。

### 安全与剪贴板

- 剪贴板自动清理时间。
- 资料显示自动遮罩时间。
- DPAPI 和备份限制说明。
- 打开应用数据目录。

### 关于

- 应用版本。
- WebView2 和数据库版本。
- 查看应用日志目录。
- 开源许可。

设置采用即时保存时，控件旁显示短暂“已保存”；需要重启生效的配置明确标识并集中提示。

## 12. 全局搜索与命令面板

提供 `Ctrl+K` 命令面板，作为效率增强功能：

- 搜索项目名称和路径。
- 打开项目详情。
- 直接为项目启动 CMD、Codex、Claude Code。
- 跳转资料库和设置。

命令面板不搜索服务器资料正文，也不显示密码内容。搜索结果分组展示，键盘上下选择、Enter 执行、Esc 关闭。

## 13. 键盘快捷键

| 快捷键 | 操作 |
|---|---|
| `Ctrl+K` | 打开全局命令面板 |
| `Ctrl+N` | 在项目页添加项目；在资料库新建资料 |
| `Ctrl+R` | 刷新当前项目文件树或项目日志 |
| `Ctrl+Shift+T` | 为当前项目打开默认终端 |
| `Ctrl+Shift+C` | 为当前项目启动 Codex |
| `Ctrl+Shift+L` | 为当前项目启动 Claude Code |
| `Ctrl+,` | 打开设置 |
| `Alt+Left` | 返回上一页 |
| `F2` | 修改当前项目展示名称，仅在项目上下文生效 |
| `Esc` | 关闭菜单、弹窗或退出当前临时状态 |

快捷键必须允许用户修改，并在与系统或其他应用冲突时提示。资料显示、永久清除等敏感操作不设置单键快捷方式。

## 14. 反馈、加载和错误设计

### 14.1 Toast

适用于：

- 路径已复制。
- 项目已恢复。
- 设置已保存。
- 文件已复制到剪贴板。

同一动作短时间重复时合并，不堆积大量通知。

### 14.2 页面内错误

适用于：

- 项目目录不可用。
- 日志格式异常。
- 数据库初始化失败。
- 资料无法解密。

错误区域包含：发生了什么、可能原因、用户可以做什么、可复制的非敏感错误码。技术详情折叠显示。

### 14.3 加载状态

- 表格初次加载使用骨架屏。
- 文件树节点使用节点内旋转图标。
- 日志同步显示阶段和数量，如“正在索引 320/10,000”。
- DPAPI 保存使用按钮内加载状态。
- 超过 300 ms 才显示明显加载动画，减少闪烁。

### 14.4 确认对话框

确认框标题必须描述实际动作：

- “从系统中移除 3 个项目？”
- “永久清除服务器资料？”
- “放弃未保存的修改？”

主要按钮使用与动作一致的动词，不使用模糊的“确定”。

## 15. 空状态设计

| 页面 | 空状态主文案 | 主操作 |
|---|---|---|
| 项目 | 尚未添加项目 | 添加第一个项目 |
| 收藏 | 还没有收藏项目 | 查看全部项目 |
| 最近使用 | 暂无最近打开记录 | 查看全部项目 |
| 项目活动 | 还没有项目日志 | 查看日志规则 |
| 服务器资料 | 尚未保存服务器资料 | 新建资料/导入 TXT |
| 回收站 | 回收站为空 | 返回有效列表 |

空状态插图保持简单，可优先使用图标和线框图形，不使用大面积装饰图占据工作区。

## 16. 无障碍与可用性

- 所有操作可通过键盘到达。
- 焦点轮廓清晰，不因美化全局移除 outline。
- 图标按钮提供可见 Tooltip 和无障碍名称。
- 文本与背景对比度满足 WCAG AA。
- 状态不能只用颜色表达，同时使用图标和文字。
- 表格选择、树节点展开和页签状态对辅助技术可识别。
- 动画遵循系统“减少动态效果”设置。
- 密码遮罩不能让屏幕阅读器直接读出正文；显示正文时给出状态提示。
- 不拦截 Windows 常规复制、粘贴和文本选择快捷键。

## 17. 窄窗口适配

### 大于等于 1280 px

- 导航展开。
- 项目文件树和预览并列。
- 活动时间线和详情并列。
- 资料库列表和详情并列。

### 1024–1279 px

- 导航默认折叠。
- 项目文件树保持 260 px。
- 活动日志详情改为点击后右侧抽屉。
- 资料库详情仍并列，但列表缩至 280 px。

### 小于 1024 px

不作为正式支持尺寸。窗口应阻止缩小到无法操作的范围，而不是把桌面工作台强行压缩成移动布局。

- 危险确认框使用统一组件，确保文案和按钮顺序一致。

## 19. 前端状态设计

建议 store：

- `useProjectStore`：项目列表、筛选、收藏、软移除。
- `useProjectDetailStore`：当前项目、页签和目录状态。
- `useFileTreeStore`：节点缓存、展开状态和错误。
- `useActivityStore`：日志索引、筛选、热力图数据和同步状态。
- `useVaultStore`：资料元数据列表；明文只保存在非持久化、短生命周期状态。
- `useSettingsStore`：非敏感设置。
- `useAppStore`：导航、窗口和全局任务状态。

所有异步状态统一至少包含：

```ts
type AsyncState = "idle" | "loading" | "success" | "error";
```

防止使用多个互相矛盾的布尔值表达同一请求状态。

## 20. 设计交付顺序

Claude 实施前端时按以下顺序完成：

1. 视觉令牌、应用框架和路由。
2. 项目列表和添加项目流程。
3. 项目详情头部与文件工作区。
4. Markdown/图片预览。
5. 活动热力图、时间线和日志详情。
6. 服务器资料库及安全交互。
7. 回收站和设置。
8. 命令面板、快捷键和桌面细节。
9. 暗色主题、窄窗口和无障碍完善。
10. 空状态、错误状态和视觉回归检查。

每个页面必须同时完成正常、加载、空、错误和不可用状态，不能只实现理想数据下的静态界面。

## 21. 设计验收清单

- [ ] 启动后两次点击内能为任一可见项目打开 CMD、Codex 或 Claude Code。
- [ ] 项目路径在列表和详情中均可识别、复制和查看完整值。
- [ ] “从系统移除”不会被误解为删除磁盘项目。
- [ ] 文件工作区没有移动和删除入口。
- [ ] 文件树只在展开时加载，并有节点级错误反馈。
- [ ] Markdown、SVG 和日志详情使用统一安全预览。
- [ ] 活动页明确显示上次同步时间，不暗示实时监听。
- [ ] 服务器资料列表不展示正文摘要。
- [ ] 选中服务器资料后正文仍默认遮罩。
- [ ] 复制资料后的剪贴板清理状态可见。
- [ ] 永久清除与软移除有明显视觉和文案差异。
- [ ] 所有页面具备加载、空、错误状态。
- [ ] 键盘可完成项目搜索、导航和常用启动操作。
- [ ] 亮色和暗色主题均满足文字对比度要求。

- [ ] 前端状态和持久化存储中不存在服务器资料正文。

## 22. 最终设计结论

本系统应表现为一个紧凑、可信赖的本地开发工作台：

- 首页用表格提高项目查找和启动效率。
- 项目详情通过“文件/活动”双工作区降低信息拥挤。
- 文件预览保持只读，保护项目文件安全边界。
- 活动热力图服务于真实任务日志，不包装成泛化统计仪表盘。
- 服务器资料库将安全状态直接体现在界面，而不是只依赖后端加密。
- 全局命令面板和键盘快捷键解决频繁寻找项目的核心痛点。

界面视觉上的重点不是“炫”，而是让路径、状态、启动动作和风险始终清晰，使用户在高频工作中无需反复确认自己当前位于哪个项目、将执行什么操作。
