## 2. 逐项整改记录

### AUD-P0-001：业务子路由完全没有渲染

**状态：已修复**

**根因**：路由将 `AppLayout.vue` 配置为父路由组件，但 `AppLayout` 使用 `<slot />` 而非 `<router-view />`，且每个 view 又自行包裹 `<AppLayout>`，导致 Vue Router 无处挂载子页面。

**修复方案**：将路由扁平化，移除父级 `AppLayout` 路由包裹，保留每个页面内部已有的 `AppLayout` 组件。

**改动文件**：`src/router/index.ts`

**验证**：新增 10 个路由测试（`src/__tests__/router.test.ts`）断言所有路由扁平配置、无父级嵌套、均有 title meta。新增 5 个组件测试（`src/__tests__/ProjectListView.test.ts`）断言"添加项目"和"添加第一个项目"在渲染后可见。

## 6. 仍存在的限制

> 当前交付范围：仅限项目所有者在本机自用，不对外分发。

1. **安装包未签名（已接受的信息项，不是当前缺陷）**：Authenticode 状态为 `NotSigned`。本机自用无需购买或配置代码签名证书，也不作为本项目验收和使用门槛；只有未来对外分发时再评估签名。
2. **品牌图带棋盘格背景**（AUD-P2-012）：当前 PNG/ICO 的棋盘格是图像内容而非透明通道，后续如在意视觉效果，可使用透明 RGBA 源图重新生成。
3. **无 Git 历史**（AUD-P2-013）：当前目录不是 Git 仓库，不影响本机运行，但会降低改动追踪和回滚能力。
4. **实际 Windows 验收**：仍需执行《验收测试方案》的关键 P0/P1 用户流程；由于使用范围仅为本机，至少完成当前 Windows 设备验收即可，Windows 10/11 双环境不作为强制门槛。
5. **前端 chunk 大小**：ProjectDetailView 因包含 ECharts 较大（1.2MB），可通过 manualChunks 后续优化，不阻断本机使用。
