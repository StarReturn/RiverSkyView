## 二、已完成整改

### 1. 活动热力图

- 只在容器拥有有效可见宽度后初始化 ECharts。
- 始终生成完整 365 天数据，空项目不再缩成左侧七个小方块。
- 根据容器宽度计算单元格尺寸。
- 使用 `ResizeObserver` 响应标签页显示、窗口大小和 DPI 布局变化。
- 组件卸载时断开监听并释放图表实例。

### 2. 空状态按钮

- 将大号空状态主图标选择器限制为直接子元素。
- 按钮内部图标固定为 16×16，禁止继承 48px 全局尺寸。
- 项目空状态“添加第一个项目”按钮增加组件级尺寸隔离。

### 3. 服务器资料按住显示

- 使用 `pointerdown/up/cancel/leave` 统一鼠标、触控笔和触摸输入。
- 支持 Space/Enter 按住显示、松开隐藏。
- 窗口失焦、全局松手、组件卸载时强制清除明文。
- 使用 generation 防止“用户已松手但异步解密随后返回”导致重新显示。
- 新增自动化测试确认松手后 `selectedContent` 被清空。

### 4. 收藏入口合并

- 侧栏不再显示独立“收藏”菜单。
- 收藏保留在项目页“全部/收藏/最近”筛选中。
- `/favorites` 保留兼容重定向，避免旧入口失效。
- 最近筛选补充真实的最近打开/活动排序。

## 三、编辑器与 IDE 子系统

### 内置支持

- Visual Studio Code
- VS Code Insiders
- Cursor
- Visual Studio 2022（`vswhere`）
- IntelliJ IDEA、WebStorm、PyCharm、Rider、CLion、GoLand、RustRover
- Android Studio
- 自定义 `.exe` / `.com`

### 用户能力

- 项目列表和项目详情共享“编辑器”分裂按钮。
- 主按钮使用项目默认或全局默认编辑器。
- 菜单显示已安装/未检测状态、可执行文件路径。
- 单个项目可设置独立默认编辑器。
- Visual Studio/Rider 自动发现 `.slnx/.sln/.csproj/.fsproj/.vbproj/.vcxproj`。
- 多目标时弹出选择，可记住项目目标。
- 设置页支持全局默认、窗口模式、重新检测和自定义配置 CRUD。

### 安全约束

- 全部启动通过 `Command::new(...).args(...)`，不拼接 Shell 命令。
- 自定义编辑器仅允许存在的绝对 `.exe` / `.com`。
- 拒绝 `.cmd/.bat/.ps1` 和 PATH 中的脚本包装器。
- 仅允许 `{projectPath}`、`{projectName}`、`{targetPath}` 三个占位符。
- 项目目标必须通过 PathGuard，拒绝 `..`、绝对路径和符号链接逃逸。
- 中文、空格、`&`、`#`、括号路径保持为一个参数。
