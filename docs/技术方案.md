# 本地项目桌面管理系统技术方案

> 文档状态：技术基线 1.0  
> 目标平台：Windows 10/11 x64  
> 更新日期：2026-07-15

## 1. 项目定位

本系统是一个以本地项目为中心的 Windows 桌面工作台，解决项目分散在不同目录、频繁进入目录、重复打开终端和启动 CLI 智能体的问题。

系统同时提供以下能力：

- 本地项目登记、搜索、重命名、软移除和恢复。
- 项目文件树、文件复制、路径复制和资源管理器入口。
- CMD、Codex、Claude Code 等终端启动入口。
- Markdown 与图片安全预览。
- 由 Codex/Claude Code 主动生成的项目工作日志。
- GitHub 风格项目活动热力图和日志时间线。
- 使用 Windows 原生加密能力保护的服务器资料库。

系统不是通用文件管理器，不提供项目文件移动和删除能力；系统也不通过实时监听整个项目目录推断工作内容。

## 2. 已确定的技术决策

### 2.1 技术栈

| 层级 | 技术 | 选型说明 |
|---|---|---|
| 操作系统 | Windows 10/11 x64 | 第一阶段只支持 Windows |
| Node.js | Node.js 24 LTS | 前端构建环境，使用 `.nvmrc` 固定主版本 |
| 包管理器 | pnpm | 使用 `packageManager` 字段和锁文件固定版本 |
| 桌面容器 | Tauri 2 | 窗口、托盘、单实例、安装包和原生调用 |
| 前端 | Vue 3 + TypeScript + Vite | 用户界面与交互 |
| UI 组件 | Element Plus | 表格、树、菜单、表单和弹窗 |
| 状态管理 | Pinia | 项目、日志、文件树和设置状态 |
| 路由 | Vue Router | 页面路由 |
| 图表 | Apache ECharts | 活动热力图与统计图 |
| Markdown | markdown-it + DOMPurify | Markdown 渲染和 HTML 净化 |
| 原生业务层 | Rust stable | 文件、终端、SQLite、DPAPI 和路径安全 |
| 数据库 | SQLite + SQLx | 本机结构化数据和迁移 |
| Windows API | `windows` crate | DPAPI、剪贴板等原生能力 |
| 日志 | `tracing` + Tauri Log | 应用诊断日志，不记录敏感正文 |
| 测试 | Vitest + Rust tests + WebDriver/人工验收 | 分层验证 |
| 安装包 | Tauri NSIS | 输出 Windows `.exe` 安装程序 |

### 2.2 明确不使用

- 不使用 FastAPI、Uvicorn 或任何本地 HTTP 服务。
- 不使用 Spring Boot 或 JVM。
- 不使用 MySQL、PostgreSQL 等常驻数据库服务。
- 不使用 Electron。
- 不实时监听项目目录。
- 不把密码、令牌、私钥或服务器资料明文写入 SQLite、日志或浏览器存储。

### 2.3 调用边界

```text
Vue 页面
  │
  │ Tauri invoke / event
  ▼
Rust 命令与服务层
  ├── ProjectService
  ├── InstructionFileService
  ├── FileService
  ├── TerminalService
  ├── ProjectLogService
  ├── VaultService
  └── SettingsService
       │
       ├── SQLite
       ├── 项目目录
       ├── Windows Terminal
       └── Windows DPAPI
```

前端不得直接获得任意 SQL 执行能力，也不得拼接 Shell 字符串。所有文件路径、进程参数和加密操作必须在 Rust 层校验并执行。

## 3. 功能范围

### 3.1 项目管理

项目列表展示：

- 项目名称。
- 项目绝对路径。
- 添加时间。
- 最近活动时间。
- 目录可用状态。
- 快捷操作。

支持：

- 通过系统目录选择器添加项目。
- 填写和修改展示名称。
- 按名称或路径搜索。
- 收藏和最近使用排序。
- 单个或批量软移除。
- 从回收站恢复项目。
- 已移除项目可以重新添加，但同一有效项目不能重复添加。

`最近活动时间`定义为最新一条有效项目日志的完成时间；没有日志时显示“暂无活动”。它不代表项目目录的文件系统修改时间。

### 3.2 项目标识与重复判断

添加项目时在根目录创建：

```text
.project-manager.json
```

示例：

```json
{
  "schemaVersion": 1,
  "projectId": "uuid",
  "createdAt": "2026-07-15T10:00:00+08:00"
}
```

标识文件只用于辅助重连和数据库恢复，不作为唯一防重复依据。防重复规则为：

1. 对用户选择路径执行绝对化和规范化。
2. 解析可以安全解析的目录联接和符号链接。
3. 按 Windows 路径大小写不敏感规则生成 `canonical_path`。
4. SQLite 对未移除项目的 `canonical_path` 建立唯一约束。
5. 已软移除的相同项目再次添加时优先提示恢复原记录。

如果项目目录只读，添加操作失败并明确说明无法创建项目管理文件，不保留半完成数据库记录。

### 3.3 智能体指令文件

添加项目时创建或更新：

```text
AGENTS.md
CLAUDE.md
pm_log/
```

- Codex 使用项目根目录的 `AGENTS.md`。
- Claude Code 使用项目根目录的 `CLAUDE.md`。
- 现有文件不得覆盖。
- 系统只管理带有明确边界标记的日志规则块。

管理块边界：

```markdown
<!-- project-manager:log-rules:start v1 -->
...
<!-- project-manager:log-rules:end -->
```

合并算法：

1. 文件不存在：以 UTF-8 创建文件并写入管理块。
2. 文件存在但没有管理块：保留原文，在末尾补足换行后追加管理块。
3. 存在同版本管理块：保持不变，操作幂等。
4. 存在旧版本管理块：只替换管理块内容。
5. 管理块损坏或只有一侧标记：停止修改并提示用户人工确认。
6. 尽量保留原文件的 UTF-8 BOM 状态和 CRLF/LF 风格。
7. 写入使用“临时文件 + 原子替换”，失败时不得留下截断文件。

规则要求智能体只在任务实际修改项目，或修改任务最终阻塞时生成日志；纯讨论、只读分析不生成日志。日志禁止包含密码、令牌、Cookie、私钥、环境变量值或完整敏感命令输出。

### 3.4 项目日志格式

目录结构：

```text
pm_log/
└── YYYY-MM-DD/
    └── HHmmss-<agent>-<short-id>.md
```

示例：

```text
pm_log/2026-07-15/143205-codex-a81f.md
pm_log/2026-07-15/151430-claude-c902.md
```

每个任务单独一个文件，避免 Claude 与 Codex 并发追加同一文件造成覆盖。文件名必须兼容 Windows，不使用冒号。

规范日志：

```markdown
---
pm_log_version: 1
agent: codex
started_at: 2026-07-15T14:20:00+08:00
finished_at: 2026-07-15T14:32:05+08:00
status: completed
---

# 完成项目列表搜索功能

## 任务目标

增加根据项目名称和路径搜索项目的能力。

## 完成内容

- 增加搜索输入框。
- 支持名称和路径模糊搜索。

## 变更文件

- `src/views/ProjectList.vue`

## 验证结果

- `pnpm test` 通过。

## 遗留问题

无。
```

允许的 `agent`：`codex`、`claude`、`other`。  
允许的 `status`：`completed`、`failed`、`blocked`。

日志同步只在以下时机发生：

- 应用启动。
- 进入项目详情。
- 用户点击“刷新日志”。
- 从应用启动的终端进程结束后，应用仍在运行时触发一次刷新。

同步过程只扫描 `pm_log`，不递归扫描整个项目。使用相对路径、文件大小、修改时间和内容哈希识别新增或变化日志。解析失败的文件显示为“格式异常”，但不得导致整个项目页面失败。

### 3.5 活动热力图与时间线

- 热力图按日志 `finished_at` 的本地日期聚合。
- 当天颜色深浅根据有效日志数量分级。
- 点击日期展示当天日志列表。
- 时间线按 `finished_at` 倒序。
- 可以按智能体和状态筛选。
- 日志详情以净化后的 Markdown 展示。
- 无法解析 `finished_at` 时使用文件修改时间并标记“推断时间”。

### 3.6 文件树与文件操作

- 文件树按需加载，展开目录时才读取直接子项。
- 默认目录优先、名称自然排序。
- 可配置是否显示隐藏文件。
- 不跟随会逃逸项目根目录的目录联接或符号链接。
- 无权限、路径过长、文件消失等错误在节点级展示。
- 禁止通过本系统移动或删除项目文件。

右键菜单：

- 文件：预览、复制文件到 Windows 剪贴板、复制绝对路径、在资源管理器中定位。
- 目录：展开/收起、复制绝对路径、在资源管理器中打开。

任何文件命令执行前都必须重新规范化路径，并验证目标仍位于登记项目根目录内。

### 3.7 文件预览

Markdown：

- 支持 `.md`、`.markdown`。
- 限制单文件预览大小，默认 2 MiB，可在设置中调整。
- 使用 markdown-it 渲染。
- 使用 DOMPurify 清除脚本、事件处理器和危险协议。
- 默认禁止远程图片和远程资源自动加载。
- 项目内相对图片经过受控资源协议加载。

图片：

- 支持 PNG、JPEG、GIF、WebP、BMP、SVG。
- 支持放大、缩小、适应窗口、原始比例、左旋、右旋和重置。
- SVG 作为不可信内容处理，禁止脚本执行。
- 超过配置上限时提示，不强制加载导致内存异常。

其他文件只显示基本信息，不把二进制内容当文本打开。

### 3.8 资源管理器和终端

打开目录使用 Windows 原生资源管理器；定位文件使用 Explorer 的选择参数。

CMD 默认命令：

```text
wt.exe -w 0 new-tab -d <project-path> cmd.exe
```

- 如果 Windows Terminal 已打开，在最近窗口创建新标签页。
- 如果没有窗口，Windows Terminal 创建窗口。
- 如果 `wt.exe` 不可用，回退为以项目目录作为工作目录启动 `cmd.exe`。
- 路径和参数以进程参数数组传递，不拼接 Shell 命令字符串。

启动配置支持：

- CMD。
- PowerShell。
- Codex。
- Claude Code。
- VS Code。
- Cursor。
- 用户自定义可执行文件和参数模板。

自定义启动配置将“可执行文件”和“参数数组”分别存储；不接受默认通过 `cmd /c` 执行任意拼接字符串。

### 3.9 服务器资料库

服务器资料库支持：

- 导入 `.txt` 文件。
- 新建、预览和编辑文本资料。
- 按名称搜索。
- 软移除、恢复和永久清除。
- 默认遮罩显示敏感内容。
- 手动复制选中文本。
- 复制后默认 30 秒清空剪贴板；仅当剪贴板仍是本应用写入内容时才清空。

安全设计：

1. 导入时读取源文件，但不修改或删除源文件。
2. 使用 Windows DPAPI CurrentUser 范围加密完整文本字节。
3. 密文写入应用数据目录，扩展名使用 `.vault`。
4. SQLite 只保存名称、标签、密文路径、时间等元数据。
5. 预览和编辑时临时解密，关闭页面后尽快清理内存缓冲区。
6. 应用日志、项目日志、错误信息和前端持久化状态不得出现资料正文。
7. 密文损坏或换 Windows 用户后无法解密时，必须显示明确错误，不得用空内容覆盖原密文。

DPAPI CurrentUser 数据通常绑定当前 Windows 登录用户及本机环境，因此普通文件复制不等同于可恢复备份。第一版备份只覆盖数据库和非敏感配置；加密资料导出/迁移功能不在本期范围内，并须在界面明确提示。

### 3.10 软删除语义

- 项目软移除：设置 `removed_at`，不删除或修改项目目录。
- 资料软移除：设置 `removed_at`，保留密文以便恢复。
- 永久清除资料：删除密文文件和数据库记录，需要二次确认。
- SSD 和日志型文件系统无法保证物理扇区级安全擦除，界面不得宣称“不可恢复安全擦除”。

### 3.11 桌面体验

- 单实例运行；再次启动时激活已有窗口。
- 支持最小化到系统托盘。
- 托盘菜单包含显示主窗口、快速项目、退出。
- 支持可配置全局快捷键唤起窗口。
- 记忆窗口大小和位置，位置失效时回到主屏幕可见区域。
- 可配置开机启动，默认关闭。
- 所有核心功能离线可用。

### 4.3 数据库约束

- 启用外键约束。
- 启用 WAL 模式。
- 所有架构变化通过 SQLx migration 管理。
- 写操作使用事务。
- 删除项目记录前不得级联删除项目目录文件。
- 日志索引以 `project_id + relative_path` 唯一。
- 密文文件写入成功后才提交资料元数据；更新资料采用新密文原子替换。


前端根据稳定错误码展示中文信息，不直接展示 Rust 调试栈。

## 7. 安全要求

- Tauri capability 按窗口和命令最小授权。
- 禁止在前端开放任意路径读写 API。
- 禁止将用户字符串拼进 Shell 命令。
- 所有项目内文件操作必须通过 `PathGuard`。
- Markdown 和 SVG 视为不可信输入。
- CSP 默认禁止远程脚本、对象和任意网络连接。
- 资料正文禁止进入 Pinia 持久化、localStorage、SQLite、日志和崩溃报告。
- 应用日志输出前经过敏感字段脱敏。
- 任何解密失败不得触发覆盖保存。
- 永久清除、批量移除等操作必须二次确认。

## 8. 性能与可用性目标

- 不递归预加载整个项目树。
- 项目列表 1,000 条记录仍可搜索和分页/虚拟滚动。
- 展开包含 5,000 个直接子项的目录时界面不得永久无响应，应显示加载状态并允许失败提示。
- 日志同步只扫描 `pm_log`，单个项目 10,000 条日志时应提供增量索引。
- 长耗时 Rust 命令异步执行，不阻塞 Tauri 主线程。
- 支持中文、空格、括号和 `#` 等常见 Windows 路径字符。
- 数据库和管理文件写入必须具备失败回滚或原子替换策略。

## 9. 测试策略
